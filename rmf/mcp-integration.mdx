## Overview

ezRMF includes a Model Context Protocol (MCP) server that exposes 30+ compliance tools to AI clients. The MCP server runs on port 8001 alongside the main API and provides per-project SSE endpoints at `/projects/:id/mcp/sse`. Connected AI clients (such as Claude Desktop or Claude Code) can analyze your ATO data, identify evidence gaps, suggest control implementations, generate documents, and export audit packages.

<Info>
The MCP server reads data from your ezRMF instance but does not make changes without user approval. All proposed changes go through the agent's ask/answer flow.
</Info>

## Available tools

The MCP server exposes 30+ tools organized by function:

### Control tools

| Tool | Description |
|------|-------------|
| `list_controls` | List all controls in a project with status and CCI counts |
| `get_control` | Get detailed control information including implementation statement |
| `update_control_status` | Propose a control status update (requires user approval) |
| `map_controls` | Map extracted text to NIST 800-53 controls |
| `get_baselines` | List available control baselines |

### Evidence tools

| Tool | Description |
|------|-------------|
| `list_evidence` | List all evidence artifacts in a project |
| `get_evidence` | Get evidence details and linked controls |
| `classify_evidence` | Classify an evidence file by type, phase, and controls |
| `link_evidence` | Link evidence to controls |
| `analyze_evidence_gaps` | Identify controls with missing or insufficient evidence |

### NIST catalog tools

| Tool | Description |
|------|-------------|
| `search_nist_catalog` | Search the NIST 800-53 Rev 5 catalog by keyword |
| `get_nist_control` | Get full NIST control details and assessment objectives |
| `get_cci_mappings` | Get CCI-to-control mappings |
| `get_control_family` | List all controls in a family |

### Document generation tools

| Tool | Description |
|------|-------------|
| `generate_document` | Generate SSPs, SARs, SCTMs, policies, and SOPs |
| `suggest_implementations` | Draft implementation statements for controls |
| `generate_policies` | Create policy documents mapped to control families |

### Project tools

| Tool | Description |
|------|-------------|
| `get_project` | Get project details and status |
| `get_inventory` | List hardware and software inventory items |
| `get_stakeholders` | List team members and roles |
| `get_system_info` | Get system characteristics and description |
| `get_pipeline` | Get current RMF phase and history |

### POAM tools

| Tool | Description |
|------|-------------|
| `list_poam` | List all POAM items |
| `create_poam` | Propose a new POAM item (requires user approval) |
| `get_poam` | Get POAM item details and milestones |

### Export tools

| Tool | Description |
|------|-------------|
| `export_emass` | Generate eMASS-formatted Excel export |
| `export_cci_results` | Generate per-CCI compliance test results |

### Integration tools

| Tool | Description |
|------|-------------|
| `get_integrations` | List active integrations and sync status |
| `get_findings` | List findings from AWS Security Hub and GuardDuty |

## Connect Claude Desktop

To connect Claude Desktop to the ezRMF MCP server, add the server configuration to your Claude Desktop settings.

### 1. Locate the configuration file

The Claude Desktop configuration file is located at:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Linux:** `~/.config/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

### 2. Add the MCP server

Add the ezRMF MCP server to the `mcpServers` section:

```json
{
  "mcpServers": {
    "ezrmf": {
      "url": "https://rmf.icbm.dev:8001/projects/{project-id}/mcp/sse",
      "headers": {
        "Authorization": "Bearer YOUR_API_TOKEN"
      }
    }
  }
}
```

Replace `{project-id}` with your project ID.

<Warning>
The API token must have at least `read` scope. Generate a token from the ezRMF UI or API — see [API reference](/rmf/api-reference) for instructions.
</Warning>

### 3. Restart Claude Desktop

After saving the configuration, restart Claude Desktop. The ezRMF tools appear in the tool list when you start a new conversation.

## Connect Claude Code

Add the ezRMF MCP server to your Claude Code settings:

```json
{
  "mcpServers": {
    "ezrmf": {
      "url": "https://rmf.icbm.dev:8001/projects/{project-id}/mcp/sse",
      "headers": {
        "Authorization": "Bearer YOUR_API_TOKEN"
      }
    }
  }
}
```

## Security considerations

The MCP server operates under the same authentication and authorization framework as the main ezRMF API:

- All MCP requests require a valid API token with appropriate scopes
- The MCP server inherits the user's role-based permissions — it can only access projects the user is authorized to view
- Generated content is advisory only and requires user approval before applying changes
- All MCP interactions are logged in the audit trail

<Tip>
Use the MCP integration to accelerate your ATO process, but always have qualified security personnel review AI-generated implementation statements, policies, and gap analyses before including them in your authorization package.
</Tip>
