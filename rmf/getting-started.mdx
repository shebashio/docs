## Prerequisites

Before deploying ezRMF, ensure you have the following:

| Requirement | Details |
|------------|---------|
| **Docker** | Docker Engine 24+ and Docker Compose v2 |
| **Node.js** | Node.js 18+ (backend runtime) |
| **OIDC provider** | Ping Identity, Cloudflare Access, or any OIDC-compatible provider |
| **Network access** | Access to the DEDZED platform via [Kasm session](/kasm-workspaces/working-within-kasm) |

<Info>
ezRMF supports multiple authentication providers including Ping Identity, Cloudflare Access, and generic OIDC/SSO. If you do not yet have credentials configured, see [Before you begin](/getting-started/before-you-begin) for enrollment instructions.
</Info>

## Deploy with Docker Compose

ezRMF ships as a Docker Compose stack. Clone the repository and configure your environment variables before starting the services.

### 1. Clone the repository

```bash
git clone https://github.com/shebashio/ezrmf.git
cd ezrmf
```

### 2. Configure environment variables

Copy the example environment file and update it with your values:

```bash
cp .env.example .env
```

Edit `.env` with the following required variables:

```bash
# Database
DATABASE_URL=postgresql://ezrmf:ezrmf_password@db:5432/ezrmf

# File storage (MinIO)
MINIO_ENDPOINT=minio
MINIO_PORT=9000
MINIO_ACCESS_KEY=your-access-key
MINIO_SECRET_KEY=your-secret-key

# OIDC Authentication
OIDC_DISCOVERY_URL=https://auth.pingone.com/<environment-id>/as/.well-known/openid-configuration
OIDC_CLIENT_ID=your-client-id
OIDC_CLIENT_SECRET=your-client-secret

# AI Agent
ANTHROPIC_API_KEY=your-anthropic-api-key

# Activity tracking
ACTIVITY_RETENTION_DAYS=365
```

<Warning>
Never commit your `.env` file to version control. The file contains secrets including your OIDC client credentials, database passwords, and API keys.
</Warning>

### 3. Start the services

```bash
docker compose up -d
```

This starts the following containers:

| Container | Port | Purpose |
|-----------|------|---------|
| `ezrmf-api` | 8000 | Express application server (REST API) |
| `ezrmf-mcp` | 8001 | MCP server for AI tool integration |
| `ezrmf-db` | 5432 | PostgreSQL 16 database with pgvector |
| `ezrmf-minio` | 9000 | MinIO object storage (S3-compatible) |

### 4. Verify the deployment

Check that all services are running:

```bash
docker compose ps
```

Verify the API is responding:

```bash
curl http://localhost:8000/api/v1/health
```

You should receive a `200 OK` response with the service status.

## Configure authentication

ezRMF authenticates users through OIDC. You can use Ping Identity, Cloudflare Access, or any OIDC-compatible provider.

### Option A: Ping Identity

In your Ping Identity admin console:

1. Navigate to **Applications** and click **Add Application**
2. Select **OIDC Web App** as the application type
3. Set the **Redirect URI** to `https://rmf.icbm.dev/auth/callback`
4. Enable the following scopes: `openid`, `profile`, `email`
5. Save the application and note the **Client ID** and **Client Secret**

### Option B: Cloudflare Access

Configure Cloudflare Access with an OIDC application pointing to your ezRMF instance. Set the redirect URI to `https://rmf.icbm.dev/auth/callback`.

### Option C: Generic OIDC / SSO

Any OIDC-compliant identity provider works. Set the `OIDC_DISCOVERY_URL`, `OIDC_CLIENT_ID`, and `OIDC_CLIENT_SECRET` in your `.env` file.

### Configure role mapping

ezRMF maps identity provider groups to ezRMF roles. Create the following groups and assign users:

| Identity provider group | ezRMF role | Description |
|-------------------------|------------|-------------|
| `rmf-pm` | PM | Program Manager |
| `rmf-issm` | ISSM | Information System Security Manager |
| `rmf-isso` | ISSO | Information System Security Officer |
| `rmf-sca` | SCA | Security Control Assessor |
| `rmf-scar` | SCAR | Security Control Assessor Representative |
| `rmf-ao` | AO | Authorizing Official |
| `rmf-engineer` | Engineer | Security Engineer |

After updating environment variables, restart the API:

```bash
docker compose restart ezrmf-api
```

## Create your first project

Once ezRMF is deployed and authentication is configured, you can create your first ATO project.

### 1. Log in to ezRMF

Navigate to `https://rmf.icbm.dev` in your Kasm browser session. You are redirected to your identity provider for authentication. After logging in, you land on the ezRMF dashboard.

### 2. Create a project

1. Click **New Project** on the dashboard
2. Enter a project name (e.g., "Mission App ATO")
3. Select the impact level: **Low**, **Moderate**, or **High**
4. Set the target ATO date
5. Click **Create**

ezRMF automatically populates the project with the appropriate NIST 800-53 control baseline based on the selected impact level.

### 3. Assign team members

1. Open the project and navigate to the **Team** panel
2. Click **Add Member**
3. Search for users by name or email
4. Assign each team member a role (PM, ISSM, ISSO, SCA, SCAR, AO, or Engineer)
5. Click **Save**

<Tip>
Assign at least one user to each role to enable the full evidence approval workflow. See [Roles and permissions](/rmf/roles-and-permissions) for details on what each role can do.
</Tip>

### 4. Import your SSP

Use the AI agent to import an existing System Security Plan:

1. Open the project **Chat** panel
2. Upload your SSP document (DOCX or PDF)
3. Ask the agent to parse and extract control implementations
4. Review the agent's proposed control mappings
5. Approve or edit the mappings

The agent reads the document, extracts control implementation statements, maps them to NIST 800-53 controls and DoD CCIs, and populates your compliance workspace.

### 5. Review the control baseline

Navigate to the **Controls** panel to see the automatically selected controls. From here you can:

- View control details and implementation guidance
- Mark controls as **Implemented**, **Partially Implemented**, or **Not Implemented**
- Add implementation statements describing how each control is satisfied
- Adjust Organization-Defined Parameters (ODPs)

### 6. Upload your first evidence artifact

1. Navigate to the **BoE** (Body of Evidence) panel
2. Click **Upload Evidence**
3. Select a file (e.g., a CKL export from STIGMATE, a policy document, or a screenshot)
4. Add a description and select the control(s) this evidence supports
5. Click **Upload**

The evidence enters the approval workflow and routes to the ISSM for review. You can preview uploaded files directly in the browser.
