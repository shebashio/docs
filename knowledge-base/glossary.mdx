| Term | Definition |
|------|-----------|
| **ATO** | Authority to Operate. Formal security approval required for federal information systems, granted by an Authorizing Official after completing the RMF process. |
| **AVA** | AWS Verified Access. Network security service that authenticates every connection without requiring a VPN. See [Zero trust access](/knowledge-base/zero-trust). |
| **Big Bang** | The DevSecOps Kubernetes platform deployed on DEDZED ephemeral clusters. Provides a pre-configured set of security and development tools on top of Kubernetes. |
| **CAC** | Common Access Card. A smart card used for initial authentication to the DEDZED platform. Required for first sign-in at `kasm.dedzed.blacklabel.mil`. |
| **CCI** | Control Correlation Identifier. Bridges NIST 800-53 controls to STIG technical checks, providing traceability from policy to implementation. |
| **CKL** | Checklist file format used by DISA STIG Viewer. STIGMATE exports scan results in CKL format for import into STIG Viewer, eMASS, or ezRMF evidence. See [CKL export](/stigmate/ckl-export). |
| **DEDZED** | The platform name (proper noun). A secure Impact Level 5 environment for rapid software development on ephemeral Kubernetes clusters. See [What is DEDZED?](/knowledge-base/what-is-dedzed). |
| **DEDZED Command Dashboard** | The main portal UI at [dashboard.dedzed.blacklabel.mil](https://dashboard.dedzed.blacklabel.mil). Used to deploy clusters, manage environments, and access platform services. |
| **Ephemeral environment** | A temporary, on-demand Kubernetes cluster with an automatic time-to-live (TTL). All data is destroyed when the TTL expires. See [Ephemeral environments](/knowledge-base/ephemeral-environments). |
| **Kasm** | The Virtual Desktop Infrastructure (VDI) that provides browser-based desktops. All DEDZED development work happens inside Kasm sessions. See [Working within Kasm](/kasm-workspaces/working-within-kasm). |
| **NIST 800-53** | NIST Special Publication 800-53 Revision 5. The federal security control framework with 20 control families used by ezRMF for ATO assessments. |
| **OIDC** | OpenID Connect. The authentication protocol used by Ping Identity for single sign-on across DEDZED services. |
| **Operator State Lock** | A Terraform-style state locking mechanism for multi-operator coordination on shared DEDZED infrastructure. See [Operator State Lock](/operator-state-lock/index). |
| **Ping Identity** | The authentication provider for the DEDZED platform. Handles CAC-based and username/password sign-in via OIDC. |
| **RBAC** | Role-Based Access Control. Used by ezRMF to define seven roles (PM, ISSM, ISSO, SCA, SCAR, AO, Engineer) with specific permissions per project. See [Roles and permissions](/rmf/roles-and-permissions). |
| **RMF** | Risk Management Framework. A NIST-defined process for managing security risk in federal systems. See [ezRMF](/rmf/index). |
| **ezRMF** | AI-powered NIST RMF compliance platform. Automates control mapping, evidence management, POAM tracking, and eMASS export with a sandboxed AI agent. See [ezRMF](/rmf/index). |
| **SBOM** | Software Bill of Materials. A machine-readable inventory of software components and dependencies, generated by Shredder during security scans. |
| **Shredder** | The DEDZED multi-engine security scanning platform. Aggregates results from Grype, Trivy, Semgrep, SonarQube, and other engines with quality gates and AI analysis. See [Shredder](/knowledge-base/shredder). |
| **STIG** | Security Technical Implementation Guide. DISA-published configuration standards for DoD environments. STIGMATE automates STIG compliance checks. |
| **STIGMATE** | AI-powered STIG compliance scanner that uses Claude agents for automated security assessments against 395+ benchmarks. See [STIGMATE](/stigmate/index). |
| **TTL** | Time to Live. The configured lifespan of an ephemeral cluster, after which it is automatically destroyed. |
| **XCCDF** | Extensible Configuration Checklist Description Format. The XML standard used by STIGs to define security checks. STIGMATE parses XCCDF files to execute automated scans. |
| **Zero trust** | A security model where no user or connection is implicitly trusted. DEDZED enforces zero trust via AWS Verified Access and dual-ingress Kasm architecture. See [Zero trust access](/knowledge-base/zero-trust). |
