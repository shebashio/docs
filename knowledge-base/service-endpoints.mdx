## Platform services

| Service | URL | Access |
|---------|-----|--------|
| DEDZED Command Dashboard | [dedzed.dedzed.blacklabel.mil](https://dedzed.dedzed.blacklabel.mil) | Browser via AWS Verified Access |
| Kasm VDI | [kasm.dedzed.blacklabel.mil](https://kasm.dedzed.blacklabel.mil) | Browser via AWS Verified Access |
| Kasm WebSocket | `kasm-direct.dedzed.blacklabel.mil` | Internal — WAF-protected WebSocket endpoint |

## Internal services

These services are accessible only from within a Kasm VDI session.

| Service | URL | Purpose |
|---------|-----|---------|
| Shredder | [shredder.dedzed.blacklabel.mil](https://shredder.dedzed.blacklabel.mil) | Multi-engine security scanning with quality gates |
| STIGMATE | [stigmate.dedzed.blacklabel.mil](https://stigmate.dedzed.blacklabel.mil) | AI-powered STIG compliance scanning |
| ezRMF | [rmf.dedzed.blacklabel.mil](https://rmf.dedzed.blacklabel.mil) | AI-powered NIST RMF compliance and ATO tracking |
| ezRMF API | `rmf.dedzed.blacklabel.mil/api` | RESTful API for ezRMF data |
| ezRMF MCP | `rmf.dedzed.blacklabel.mil:8001/projects/:id/mcp/sse` | Model Context Protocol server with 30+ compliance tools |

## Authentication

All services authenticate through **Ping Identity**. Initial sign-in requires a valid CAC at `kasm.dedzed.blacklabel.mil`. Within Kasm sessions, use your DEDZED username and password.

## Network access

DEDZED uses a [zero trust access model](/knowledge-base/zero-trust) — no VPN is required. All external access flows through AWS Verified Access, which authenticates every connection before granting access to platform services.

<Info>
Internal services (Shredder, STIGMATE, ezRMF) are not accessible from outside a Kasm session. Start a Kasm VDI session first, then navigate to the service URL within the session browser.
</Info>
