## Email

The best way to report an issue or request help is by emailing the DEDZED team at [dedzed-tech@shebash.io](mailto:dedzed-tech@shebash.io).

## Issue severity

When reporting an issue, select the severity that best matches your situation. This helps the team prioritize and respond appropriately.

| Severity | Description | Examples |
|----------|-------------|---------|
| **Critical** | Platform unusable — all users affected or data at risk | Kasm VDI unreachable, DEDZED Command Dashboard down, authentication system failure |
| **High** | Major feature broken — your work is blocked | Cluster deployment fails, cannot connect to a running cluster, Shredder scans not completing |
| **Medium** | Feature degraded — workaround available | Slow provisioning, intermittent WebSocket disconnects, dashboard UI rendering issues |
| **Low** | Minor issue or question — no immediate impact | Documentation clarification, feature request, cosmetic UI issue |

## What to include in a report

When reporting an issue, include the following details so the team can help you quickly:

- **Severity level** from the table above
- A description of what you were trying to do
- The steps to reproduce the issue
- Any error messages or screenshots
- Your cluster name, if applicable
- The service affected (Kasm, DEDZED Command Dashboard, Shredder, STIGMATE, RMF)

## How issues are handled

1. **Email** — Send your report to [dedzed-tech@shebash.io](mailto:dedzed-tech@shebash.io). Reviewed during business hours.
2. **Escalation** — Critical and high severity issues are escalated to the engineering team immediately. You will receive a response acknowledging the issue.
